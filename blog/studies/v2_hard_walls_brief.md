# Study Brief: `v2-hard-walls`

## Claim

Draft claim for confirmation:

> With a real arena, wall-aware rewards, and stronger predator learning, do we get collective structure that improves survival without the runaway escape exploit?

This wording is intentionally narrower than a full emergence claim. It does not claim minimal emergence, generalization beyond the frozen setup, or mechanism identification. It is meant to answer the specific failure observed in `v1`: prey solving the game by dispersing out of the nominal cube while predators remained effective only against compact flocks.

## Frozen Environment Version

- Study ID: `v2-hard-walls`
- Config: `configs/studies/v2_hard_walls.toml`
- Environment class: `VectorMurmurationEnv`
- Boundary definition:
  - `boundary_mode = "hard_walls"`
  - hard cube boundaries enforced in physics
  - soft wall-pressure penalties still applied near walls
- Reward definition:
  - survival reward `+0.1`
  - collision penalty `-1.0` per collision neighbor
  - death penalty `-100.0`
  - PBRS boundary weight `0.0`
  - PBRS density weight `2.0`
  - wall penalty `0.5`
  - predator wall penalty `0.15`
  - predator catch reward `+10.0`
  - predator team catch reward `+1.0`
  - predator hunger penalty `-0.05`
- Curriculum definition:
  - curriculum disabled
  - predator catch radius fixed at `1.5`
- Motion definition:
  - prey base speed `5.0`
  - prey min speed `2.5`
  - predator sprint multiplier `1.5`
  - predator turn multiplier `1.5`

## Training Defaults

The main week-2 training changes relative to `v1` are:

- predator actor learning rate increased to `2e-4`
- entropy coefficient starts at `0.02`
- entropy anneals more slowly over `5000` epochs
- entropy floor remains at `0.002`
- default long-run length remains `30000` epochs

The intent is to reduce early predator under-training and avoid the quick collapse into a low-exploration prey-favored equilibrium.

## Evaluation Metrics

Every candidate run should be summarized by the fixed report generated by `analyze_run.py`.

Required scorecard:

- prey survival fraction over time
- predator capture rate per step
- nearest-neighbor distance distribution
- local-density distribution
- polarization / heading alignment
- connected-component count under a distance graph
- swarm radial spread from center of mass
- mean distance to world center
- outside-of-bounds fraction
- fringe / edge exposure fraction

The two new diagnostics for week-2 are especially important:

- `distance_to_center`
- `outside_fraction`

They exist to explicitly detect the escape exploit that invalidated the `v1` equilibrium.

## Baseline Ladder

Named baseline configs still live under `configs/experiments/baselines/`.

For week-2 review, the minimum comparison set should be:

1. `random` prey vs `random` predator under the `v2` study config
2. learned prey vs learned predator checkpoints from `v2`
3. learned prey vs random predator
4. random prey vs learned predator
5. heuristic boids prey vs learned predator

Checkpoint-backed baselines remain intentionally explicit: the config exists now, and the exact checkpoint path must be supplied at execution time.

## Seed Policy

- exploratory comparison: `2-3` seeds
- candidate benchmark claim: `5` seeds
- external review package: `8-10` seeds

Week-2 should still begin with `3` seeds for the first serious pass, then expand only after the scorecards show that the exploit is gone and the new equilibrium is qualitatively healthier.

## Exit Criteria For Week 2

- the hard-wall study version is represented by a frozen config file
- the claim can be read in one sentence from this brief
- the default training entrypoint now targets `v2`
- every run can emit the same report schema plus the new boundary diagnostics
- new runs stay inside the arena, i.e. `outside_fraction ~= 0`
